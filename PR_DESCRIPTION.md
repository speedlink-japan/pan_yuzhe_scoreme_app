# 🎮 フルスクリーンモード独立実装 & レイアウト管理の完全修正

## 📋 概要

MyScore アプリのレイアウト管理機能を大幅に改善し、フルスクリーンモードを完全に独立した機能として実装しました。通常モード（ユーザーカスタマイズ可能）とフルスクリーンモード（自動グリッドレイアウト）を明確に分離し、両モード間の状態遷移を正確に制御します。

## 🎯 主な変更内容

### 1. フルスクリーン機能の独立化
- **独立したストレージ**: `myscore_fullscreen_backup` キーで別管理
- **自動バックアップ/復元**: フルスクリーン入場時に通常レイアウトをバックアップ、退場時に復元
- **自動グリッドレイアウト**: フルスクリーン中は見えるパネルのみでグリッド配置
- **参照追跡**: `useRef` で前回のレイアウトモードを追跡し、確実な遷移制御

### 2. レイアウト初期化の改善
- **ビューポート認識**: ページ初回アクセス時に正しいウィンドウサイズに基づいてレイアウト選択
- **段階的な初期化**: localStorage → ビューポート対応デフォルト → hydration フロー

### 3. パネル表示/非表示の動作改善

#### 通常モード
- パネルON/OFF時に既存パネルのレイアウト保持
- 前回OFF時の位置情報を優先使用して復元

#### フルスクリーンモード
- パネルON/OFF時に自動的にグリッドレイアウトを再計算
- 残りパネルが自動リサイズされて最適配置される

### 4. リセット機能の修正
- パネルグループを中央寄せ配置
- リセット後のOFF→ON遷移で中央寄せ位置を完全に保持

## 🔄 状態管理フロー

```
【通常モード】
  パネルON/OFF
    ↓
  panelPositions 保持（ユーザー調整したレイアウト保存）
    ↓
  ON時に前回の位置から復元

【フルスクリーン入場】
  1. myscore_fullscreen_backup に現在の通常モードレイアウトを保存
  2. calculateDynamicLayout で見えるパネルのみグリッド配置
  3. パネルON/OFF時も常にグリッドレイアウトを再計算

【フルスクリーン退場】
  1. myscore_fullscreen_backup から通常モードレイアウトを復元
  2. 中央寄せ位置を含めて完全に復元
  3. clearFullscreenBackup でバックアップ削除
```
